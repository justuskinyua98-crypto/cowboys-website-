Cowboys Group Holdings Website Guidebook

Print Edition

Author: Justus Kinyua and team

Project: Cowboys Group Holdings website + lightweight backend

This guide records how the website was built, how it works, and how to maintain, improve, and deploy it.

Purpose

To preserve technical knowledge for the next generation, including the exact build path, key decisions, and practical workflows.

1. Technology Used

  • Frontend: HTML, CSS, Vanilla JavaScript
  • Backend: Node.js native HTTP server
  • Storage: JSON files under data/
  • Version control: Git + GitHub
  • Media: Local assets under assets/

5. How It Works

Frontend (script.js)

  • Fetches content from /api/content
  • Renders sections (outfits, events, team, socials, etc.)
  • Handles cart and checkout interactions
  • Handles admin forms and save actions

Backend (server.js)

  • Serves static site files
  • Serves API routes for content and payments
  • Stores data in JSON files
  • Accepts media uploads
  • Applies security headers and rate limits

6. Core API Endpoints

GET /api/health GET /api/content PUT /api/content (admin key required) GET /api/payments/config POST /api/payments/checkout GET /api/payments/status GET /api/payments/receipt POST /api/payments/confirm-manual (admin key required) POST /api/media (admin key required)

7. Media Workflow

  1. Place files in the right assets folder.
  2. Use clean file names (no spaces/special symbols).
  3. Map each file path in data/content.json.
  4. Hard refresh browser (Cmd+Shift+R).

Example

"photo": "assets/team/aaron-sang.jpg"

8. SEO Setup

  • Canonical and social meta tags in index.html
  • robots.txt at root
  • sitemap.xml at root
  • JSON-LD Organization schema

After deployment, submit sitemap in Google Search Console and request indexing for homepage.

9. Security Practices

  • Never expose tokens in public/chat.
  • Use strong ADMIN_KEY in production.
  • Rotate compromised tokens immediately.
  • Use HTTPS for live deployment.

10. Final Lesson

Real software is built by iteration: build, test, fix, improve, and save progress in Git.

This guide is proof that steady progress creates real systems.
